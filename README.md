# ADIE — Autonomous Design Integrity Engine
# README

## Overview

ADIE is a 4-agent Python system that runs inside Autodesk Fusion 360 as an add-in and autonomously validates CAD designs against engineering standards extracted from PDF documents.

**Developer:** Project for var  
**AI Backbone:** Gemini 1.5 Flash (free tier, google-generativeai SDK)  
**CAD Platform:** Autodesk Fusion 360 Personal Use (free)

---

## File Structure

```
adie/
├── setup/

├── validation_scripts/           # Auto-generated by Synthesizer (one .py per rule)
├── data/
│   ├── rules.json                # Output of Analyst Agent
│   ├── violations.json           # Overwritten by Monitor each run
│   ├── audit_log.json            # Append-only liability record
│   ├── pending_suggestion.json   # HITL staging area
│   ├── failed_chunks.log         # Analyst parse failures
│   ├── uncompiled_rules.log      # Synthesizer AST failures
│   └── script_errors.log         # Runtime validation errors
└── standards/                    # Input PDFs (ISO, DIN, etc.)
```

## Day-by-Day Setup

### Day 1 — Environment & Toolchain

```bash
# Create conda environment
conda create -n adie python=3.10
conda activate adie

# Install pythonocc (CONDA ONLY — never pip)
conda install -c conda-forge pythonocc-core

# Install Python packages
pip install pymupdf google-generativeai
```

**Test inside Fusion 360:**
1. Open Fusion 360
2. `Tools → Scripts and Add-ins → Scripts`
3. Click `+` → browse to `adie/setup/test_toolchain.py`
4. Click **Run**
5. Expected: modal showing body names + volumes, ✅ in console

---

### Day 2 — Analyst + Synthesizer

**Step 1 — Place your PDF:**
```
adie/standards/ISO_2768.pdf
```

**Step 2 — Run Analyst Agent:**
```bash
conda activate adie
cd adie
python setup/analyst_agent.py \
  --pdf standards/ISO_2768.pdf \
  --output data/rules.json \
  --api-key YOUR_GEMINI_API_KEY
```

Output: `data/rules.json` — array of rule objects.

**Step 3 — Run Synthesizer Agent:**
```bash
python setup/synthesizer_agent.py \
  --rules data/rules.json \
  --output validation_scripts/ \
  --api-key YOUR_GEMINI_API_KEY
```

Output: `validation_scripts/R001_min_wall_thickness.py` etc.  
Verify: every file must pass `ast.parse()` (done automatically).

---

### Day 3 — Monitor + Mitigation (Fusion Add-in)

**Step 1 — Edit config in `fusion_addin/ADIE.py`:**
```python
ADIE_CONFIG = {
    "gemini_api_key": "YOUR_GEMINI_API_KEY",
    ...
}
```
Or set environment variable: `GEMINI_API_KEY=your_key_here`

**Step 2 — Load as Fusion Add-in:**
1. `Tools → Scripts and Add-ins → Add-ins` tab
2. Click `+` → browse to `adie/fusion_addin/`
3. Select `ADIE.py`
4. Click **Run**

**Step 3 — Test:**
1. Open a Fusion design
2. Create a thin wall (< 2mm if using ISO wall thickness rule)
3. Press Ctrl+S (save)
4. Watch the TextCommandPalette — you should see a violation message
5. Click **Yes** in the dialog to approve the AI fix suggestion
6. Check `data/audit_log.json` for the record

### Day 4 — SDF Wall Thickness (Optional, requires pythonocc)

The SDF check is a geometrically rigorous alternative to the Synthesizer-generated wall thickness function.

**Enable:**
```python
ADIE_CONFIG["sdf_enabled"] = True
```

**Standalone test:**
```bash
conda activate adie
python fusion_addin/wall_thickness_sdf.py \
  --step data/step_exports/MyBody.step \
  --min-thickness 2.0
```

## Architecture: 4 Agents

| Agent | Runs Where | Gemini? | Trigger |
|-------|-----------|---------|---------|
| **1 — Analyst** | Outside Fusion (conda env) | ✅ Yes | Once at setup |
| **2 — Synthesizer** | Outside Fusion (conda env) | ✅ Yes | Once at setup |
| **3 — Monitor** | Inside Fusion (add-in) | ❌ Never | documentSaved + documentActivated |
| **4 — Mitigation** | Inside Fusion (add-in) | ✅ Yes | When Monitor finds violations |

---

## Rules JSON Schema

```json
[
  {
    "rule_id": "R001",
    "rule": "min_wall_thickness",
    "value": 2.0,
    "unit": "mm",
    "condition": "material == Aluminum",
    "source_page": 4,
    "severity": "critical"
  }
]
```

**Severity:**
- `critical` — blocks design (HITL gate fires)
- `warning` — flagged (HITL gate fires)
- `info` — logged only (Mitigation Agent skips)

---

## Non-Negotiables

1. Monitor Agent makes **zero Gemini calls** — fully deterministic
2. AI suggestions **never shown** without HITL gate confirmation
3. `audit_log.json` is **append-only** — never overwritten
4. Every Gemini JSON response validated with `json.loads()`
5. Every Synthesizer output validated with `ast.parse()`
6. Monitor debounced to **5-second minimum** between runs
7. Each validation script wrapped in `try/except` — one failure never halts Monitor
8. Only stable Fusion API: `bRepBodies`, `physicalProperties`, `BRepEdge`, `measureManager`, `exportManager`
9. All Gemini prompts specify "Return ONLY..." — no preamble, no markdown
10. `pythonocc` installed via **conda only**

---

## Gemini Free Tier Usage

- **Rate limit:** 15 RPM, 1500 requests/day
- **Analyst + Synthesizer:** run once at setup — not a runtime concern
- **Monitor:** zero Gemini calls
- **Mitigation:** max 1 call per violation per save — with 5s debounce, real-world < 50 calls/session
